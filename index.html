OSCARAgricultural Mapping UAV
Frame850mm Custom CF
Motors4114 320KV
Battery6S 5000mAh
Est. Flight25 – 30 min
Overview

OSCAR is an endurance-optimized agricultural mapping quadcopter built on a custom-designed 850mm carbon fiber frame. The entire software stack — MAVLink integration, distance-based camera trigger, geotagging, NDVI processing, and WebODM API pipeline — was written from scratch.

This is not a kit build. Every subsystem — frame, gimbal, software, and imaging pipeline — was designed, built, and coded by me.

Frame Design

The frame uses 16mm round carbon fiber tubing (2mm wall) and 3D-printed PETG-CF structural connectors. Round tube enables internal wire routing — all motor phase wires run through the hollow arm center, resulting in zero exposed wiring on any arm.

Motor mounts are round clamp-style blocks gripping 45mm of arm tube with M3 bolts through heat-set inserts. Nylon M3 bolts on center plates for electrical isolation. Steel M3 bolts on motor mounts for structural clamping load — a deliberate materials decision.

NDVI Imaging Pipeline

The payload is a Raspberry Pi Zero 2W with a NoIR Camera v2 and a 680nm bandpass filter. Rather than a time-based trigger I built a Haversine distance-based capture system — a photo fires every 5 meters of GPS ground track, accounting for speed variation, crosswind, and heading deviation. All images are geotagged in real-time from the MAVLink GPS stream over serial.

The stitching pipeline uses OpenCV to assemble hundreds of survey images into a single orthomosaic. NDVI approximation applies the formula (NIR − Red) / (NIR + Red) per pixel. A 680nm bandpass filter on the NoIR camera isolates infrared from the red channel for ~$3 instead of a $400 dedicated multispectral camera.

Overview

STRATO is a custom-built rocket with a fully self-designed avionics system. The flight computer is built around a LilyGO T-Beam V1.2 — an ESP32 board with integrated GPS, LoRa radio, and OLED display. The entire structural design was modeled in Onshape, simulated in OpenRocket, and physically built and launched.

The avionics detect flight phases in real time using a BMP280 barometric sensor and transmit live telemetry via LoRa to a ground receiver. An ESP32-CAM captures onboard video during flight.

Rocket Design

Designed with an Ogive nose cone for optimal aerodynamic efficiency and custom clipped delta fins cut from balsa wood. Fin edges were sanded to an aero profile. Stability coefficient tuned to 1.9–2.0 cal — below 3.47 (over-stable) which risks the rocket turning aggressively into wind during launch or breaking apart under high G-forces.

Final dimensions: 105cm length. Calculated apogee: ~559m. One main parachute deployment via ejection charge. Body tube livery: blue and white, inspired by BPS.space's Avalanche rocket.

Self-Designed Avionics

The avionics bay was designed from scratch in Onshape with dedicated brackets for each component. A full flight state machine runs on the ESP32: IDLE → LAUNCH_DETECT → BOOST → COAST → APOGEE → DESCENT → LANDED. State transitions are triggered by BMP280 altitude deltas.

Key problems solved during build: TX/RX wire solder bridges causing silent download failures, ESP32-CAM voltage (OV3660 runs 2.8V not 3.3V — swapped to OV2640 to match rail), and altitude averaging code to smooth ±0.3m barometric noise.

Overview

TrailBeam is a real-time GPS hiking tracker built on a LilyGO T-Beam running Meshtastic firmware. The T-Beam connects to a phone hotspot and broadcasts live GPS position over MQTT to a Python script on a laptop, which logs coordinates and plots the trail on a live Leaflet.js map. No cell coverage required.

The build is complete and trail-tested. Taken on a walk around the neighborhood with 10 satellite GPS lock, position updates every 30 seconds, and stable MQTT connection throughout.
